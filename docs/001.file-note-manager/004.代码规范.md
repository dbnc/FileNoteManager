## 一、命名规范

### 1.1 文件命名
- 每个文件包含一个主要类，文件名与类名相同
- 接口文件以 `I` 前缀：`INoteService.cs`
- ViewModel 文件以 `ViewModel` 后缀：`MainViewModel.cs`
- XAML 代码后置文件与 XAML 同名：`MainWindow.xaml.cs`

### 1.2 类命名
| 类型 | 规范 | 示例 |
|------|------|------|
| 接口 | PascalCase + I 前缀 | `INoteService` |
| 实现类 | PascalCase | `NoteService` |
| 数据库实体 | PascalCase + Entity 后缀 | `NoteEntity` |
| 数据模型 | PascalCase | `Note`, `Tag` |
| ViewModel | PascalCase + ViewModel 后缀 | `MainViewModel` |
| Repository | PascalCase + Repository 后缀 | `NoteRepository` |
| Service | PascalCase + Service 后缀 | `NoteService` |

### 1.3 成员命名
```csharp
// 私有字段：下划线前缀 + camelCase
private readonly INoteService _noteService;

// 属性：PascalCase
public string Content { get; set; }

// 方法：PascalCase
public Note GetNote(string path) { }

// CommunityToolkit.Mvvm 源生成器字段：下划线前缀 + camelCase
[ObservableProperty]
private string _searchText = string.Empty;
// 生成属性：SearchText
```

## 二、项目分层规范

### 2.1 Core 层规则
- **不依赖** UI 框架（WPF、WinForms 等）
- **不依赖** DI 容器具体实现（只允许 DI Abstractions）
- 所有数据库操作必须通过 Repository 接口
- Service 层不直接操作 `DatabaseManager`（SearchService/StatsService 除外，因需要 FTS5 特殊查询）

### 2.2 Repository 规则
- 只处理数据库 CRUD，不包含业务逻辑
- 使用 Dapper 的参数化查询，禁止字符串拼接 SQL
- 方法命名参考：`GetById`, `GetAll`, `Save`（upsert）, `Delete`

### 2.3 Service 规则
- 包含业务逻辑，不直接访问数据库
- 通过构造函数注入 Repository 依赖
- 所有 Service 必须对应一个接口

### 2.4 UI 层规则
- 所有业务逻辑放在 ViewModel，View（XAML + code-behind）只处理纯 UI 逻辑
- ViewModel 通过构造函数注入 Service
- 使用 `ObservableCollection<T>` 绑定列表数据
- 命令使用 `[RelayCommand]` 或 `RelayCommand`

## 三、异常处理规范

```csharp
// ✅ 正确：有意义的异常类型
throw new ArgumentException("Note content cannot be blank", nameof(content));
throw new FileNotFoundException($"Backup file not found: {backupPath}");

// ✅ 正确：Service 层吞掉已预期异常，记录日志
try { ... }
catch (ArgumentException ex)
{
    // Log and skip
}

// ❌ 错误：空 catch 块
try { ... } catch { }

// ❌ 错误：重新抛出异常丢失堆栈
try { ... }
catch (Exception ex) { throw ex; } // 应用 throw; 或 throw new Exception("...", ex);
```

## 四、数据库操作规范

### 4.1 参数化查询
```csharp
// ✅ 正确：使用匿名对象参数
connection.Execute(
    "INSERT INTO NoteEntity (Path, Content) VALUES (@Path, @Content)",
    new { Path = path, Content = content }
);

// ❌ 错误：字符串拼接
connection.Execute($"INSERT INTO NoteEntity VALUES ('{path}', '{content}')");
```

### 4.2 连接管理
```csharp
// ✅ 正确：使用 using 确保连接释放
using var connection = _dbManager.CreateConnection();
```

### 4.3 事务使用
- 多步操作（路径更新，同时更新 Note + NoteTag + History）必须使用事务
- 单步操作不强制要求事务

## 六、测试规范

### 6.1 测试命名
```
[被测方法名]_[测试场景]_[预期结果]
GetNote_WithExistingPath_ReturnsNote
GetNote_WithDeletedNote_ReturnsNull
CreateNote_WithBlankContent_ThrowsArgumentException
```

### 6.2 测试结构（AAA 模式）
```csharp
[Fact]
public void CreateNote_WithValidInput_SavesToRepository()
{
    // Arrange
    var mockRepo = new Mock<INoteRepository>();
    var service = new NoteService(mockRepo.Object, ...);

    // Act
    var result = service.CreateNote("C:\\test.txt", "Hello", new[] { "tag1" });

    // Assert
    Assert.NotNull(result);
    Assert.Equal("Hello", result.Content);
    mockRepo.Verify(r => r.Save(It.IsAny<NoteEntity>()), Times.Once);
}
```

### 6.3 集成测试
- 使用内存 SQLite（`:memory:`）进行集成测试
- 每个测试方法创建独立的 `DatabaseManager` 实例

```csharp
public class NoteRepositoryIntegrationTests : IDisposable
{
    private readonly DatabaseManager _dbManager;

    public NoteRepositoryIntegrationTests()
    {
        // 使用临时文件而非内存，确保FTS5支持
        _dbManager = new DatabaseManager(Path.GetTempFileName());
    }

    public void Dispose() => _dbManager.Dispose();
}
```
